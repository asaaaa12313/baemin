## 🎯 목표 4요소

| 요소 | 내용 |
|---|---|
| **목표** | ① 가짜 성공(거부됐는데 ✅) 막기 ② 정상 폴백 로그를 오류처럼 안 보이게 ③ 빈 이메일 누락 스킵 |
| **범위** | `server.py` 의 `process_single_item` + 데이터 검증부만. 다른 파일·로직 손대지 않음 |
| **종료 조건** | 멀쩡한 접수는 **절대 실패 처리 X** (사용자 1순위). 명백한 거부 신호가 보일 때만 실패. 문법 검사 PASS + 검수 봇 Critical/High 0건 |
| **검증 명령** | `python -m py_compile server.py` |

## ⭐ 핵심 설계 원칙 — "실패 안되게"

완료 화면 문구를 추측으로 박으면 멀쩡한 것도 실패될 수 있으므로, **3단 우선순위**로 안전하게:

1. **완료 신호 보이면 → 성공** (가장 확실, 거부 검사 건너뜀)
2. 완료 신호 없는데 **명백한 거부 신호만 보이면 → 실패** (가게번호 불일치 등)
3. **둘 다 애매하면 → 기존처럼 성공 유지** (정상건 실패 방지 — 최우선)

→ 완료 키워드가 거부보다 우선이라 오탐 최소화. done_kw가 실제 문구와 안 맞아도 케이스 3으로 빠져 성공 유지 = 무해.
